## 🧭 What I Build, and Why

I'm a self‑taught frontend architect who believes **great UIs start from a solid system**.  
I spend my days designing component primitives, optimising build pipelines, and occasionally reading OS source code to understand how things really work.  
I turn complex product requirements into clean, maintainable frontend systems — CMS dashboards, data‑driven interfaces, and interactive tools that scale.

My playground is the **edge between frontend and platform**: I explore OS concepts, game loops, and low‑level mental models to write exponentially simpler UI code.

> **Mindset:** Build once, scale forever.  
> **Architecture:** Modular & scalable.  
> **Code Quality:** Clean code & modern patterns.  
> **Learning:** Deep & systematic.
